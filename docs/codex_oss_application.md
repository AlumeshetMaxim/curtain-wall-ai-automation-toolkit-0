# Codex OSS Application Notes

## Project name

CW Facade Automation Toolkit

## Short description

An open-source automation toolkit for CAD, BIM, facade engineering and curtain wall workflows.

The project is focused on automating repetitive planning, drafting and production-preparation tasks such as CAD drawing processing, page setup, PDF publishing, quantity take-offs, BOM/MTO generation, DWG data extraction, glass and profile naming, STEP annotation checking and revision comparison.

## Why Codex matters for this project

Facade and curtain wall teams often work under very tight deadlines. Many tasks still require manual drawing checks, manual Excel preparation, repeated PDF publishing, drawing-package splitting, file comparison and data extraction.

Codex already helps transform tasks that previously looked too complex or too time-consuming into working scripts and automation tools. It can help engineers and planners convert real engineering knowledge into code, even when they are not full-time software developers.

## How API credits would be used

API credits would be used to build, test and improve AI-assisted tools for real construction automation workflows:

- generating and improving Python scripts for AutoCAD, BricsCAD and Rhino workflows;
- analyzing simplified CAD-related data and Excel schedules;
- automating BOM/MTO and revision comparison workflows;
- supporting PDF publishing, PDF splitting and document-package preparation;
- helping create reusable public examples for facade planners and construction engineers;
- building safe templates that keep confidential project data out of public code.

## Long-term impact

The goal is to help bring automation into facade planning and building-envelope engineering, first in Israel and later internationally. The project aims to bridge the gap between construction expertise and software development, making advanced automation more accessible to planners, engineers and drafting teams.
